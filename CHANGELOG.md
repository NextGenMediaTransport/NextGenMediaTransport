### Fixed

- **OBS input + Studio:** **BGRX → RGBA** swizzle now treats VMX decode output as **opaque** (**alpha 255**): **`VMX_DecodeBGRX`** often leaves **X = 0**, which made OBS blend the whole frame to black. See [ngmt-obs-plugin/CHANGELOG.md](ngmt-obs-plugin/CHANGELOG.md), [ngmt-studio/CHANGELOG.md](ngmt-studio/CHANGELOG.md), and [wire format — pixel format](docs/protocol/ngmt-wire-format.md#pixel-format-and-alpha-channel-additional-feature).
- **OBS input:** Stream **`gs_texture_create`** now uses **`GS_DYNAMIC`** for per-frame **`gs_texture_set_image`**; **recv FFI** no longer truncates oversized QUIC datagrams. See [ngmt-obs-plugin/CHANGELOG.md](ngmt-obs-plugin/CHANGELOG.md) and [ngmt-transport/CHANGELOG.md](ngmt-transport/CHANGELOG.md).
- **OBS input:** **`video_render`** uses **`obs_source_draw`** instead of nesting **`gs_effect_loop`** on **`OBS_EFFECT_DEFAULT`**, fixing black preview on **OBS 32.x** / macOS when logs showed **`VMX v1 frame`** plus **`gs_effect_loop: An effect is already active`**. See [ngmt-obs-plugin/CHANGELOG.md](ngmt-obs-plugin/CHANGELOG.md) and [ngmt-obs-plugin#18](https://github.com/NextGenMediaTransport/ngmt-obs-plugin/issues/18).
- **OBS input:** **`show`/`hide`** reconnect QUIC when the source becomes visible again after leaving a scene; **Refresh discovery** triggers **`obs_source_update_properties`**. See [ngmt-obs-plugin/CHANGELOG.md](ngmt-obs-plugin/CHANGELOG.md).
- **OBS input:** Manual vs Discovery visibility updates without closing properties ([ngmt-obs-plugin#19](https://github.com/NextGenMediaTransport/ngmt-obs-plugin/issues/19)); README documents QUIC bind **EADDRINUSE** when two senders share a port ([ngmt-obs-plugin#20](https://github.com/NextGenMediaTransport/ngmt-obs-plugin/issues/20)).
- **Studio Monitor (`ngmt-studio`):** **corner resize** interactors register **every frame** (egui **prev-pass** hit testing); **VU / solo** meter column **layout split** so the solo control stays inside the slot frame. See `ngmt-studio/CHANGELOG.md` and [monitor-preview-decode.md](docs/testing/monitor-preview-decode.md).
